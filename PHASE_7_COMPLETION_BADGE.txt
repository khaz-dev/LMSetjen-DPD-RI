║  HOW IT WORKS                                                            ║
║  ──────────────────────────────────────────────────────────────────────  ║
║                                                                           ║
║  1. User logs in → System detects all available roles                    ║
║                                                                           ║
║  2. User navigates to dashboard → Role indicator visible in header       ║
║                                  Shows: [Current Role ▼]                 ║
║                                  Color-coded (blue/green/red)            ║
║                                                                           ║
║  3. User clicks role badge → Dropdown opens with available roles         ║
║                              Shows current role with ✓                   ║
║                              Shows other roles as options                ║
║                                                                           ║
║  4. User selects new role → API call to /auth/select-role/              ║
║                             Loading spinner shown                        ║
║                             JWT token updated with new role              ║
║                                                                           ║
║  5. API succeeds → Toast notification shows success                      ║
║                    "Role Switched - You are now in [Role] mode"          ║
║                    Page reloads after 500ms                              ║
║                                                                           ║
║  6. Page reloads → New dashboard loads with new role                     ║
║                    RoleRoute (Phase 6) enforces new permissions          ║
║                    Headers update to show new role                       ║
║                    User can switch back anytime                          ║
║                                                                           ║
║  7. User can switch again → Same process from new role                   ║
║                                                                           ║
