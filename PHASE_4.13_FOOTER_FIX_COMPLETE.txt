WHY THIS FIXES THE PROBLEM
═════════════════════════════════════════════════════════════════════════════

ISSUE: Fragment Wrapper Problem

React Fragments (<>...</>) are special - they don't create real DOM elements:

  Fragments: <> ... </> → Renders as: (nothing, no container)
  Divs:      <div>...</div> → Renders as: <div>...</div>

This means:
  ✗ Fragments cannot use CSS flexbox
  ✗ Fragments cannot have CSS properties applied
  ✗ Fragments cannot create layout context
  ✗ Children of fragments don't get flex properties

With <div>:
  ✓ Can apply display: flex
  ✓ Can set flexDirection: column
  ✓ Can set minHeight: 100vh
  ✓ Children (section, footer) participate in flex layout

RESULT:
  UsersAdmin wasn't using flexbox → footer positioned incorrectly → height broken


ISSUE: Missing Footer CSS Margin

The 80px margin-top creates space between page content and footer:

  Without margin-top: 80px
  ┌──────────────────────────┐
  │ Footer immediately below │  ← Looks cramped, inconsistent
  │ content with no spacing  │
  └──────────────────────────┘

  With margin-top: 80px
  ┌──────────────────────────┐
  │ Content                  │
  │                          │  ← 80px breathing room
  ├──────────────────────────┤
  │ Footer with proper space │  ← Looks balanced, consistent
  └──────────────────────────┘

DashboardAdmin had this rule, making its footer look correct.
SystemDocumentation and UsersAdmin were missing it, making their footers look inconsistent.

LESSONS LEARNED
═════════════════════════════════════════════════════════════════════════════

1️⃣ FRAGMENTS FOR RENDERING, NOT LAYOUT
   ├─ Use: <> for conditional rendering, lists without wrapper
   ├─ Don't use: <> for page structure or layout containers
   └─ Always use: <div> for layout with CSS properties

2️⃣ CONSISTENCY PATTERNS
   ├─ When fixing one component, check if issue exists elsewhere
   ├─ Use one page as reference template
   ├─ Apply same structure to all similar pages
   └─ Document the pattern for future use

3️⃣ FLEXBOX BEST PRACTICES
   ├─ Root: display: flex, flexDirection: column, minHeight: 100vh
   ├─ Content: flex: 1 (expands to fill available space)
   ├─ Footer: margin-top for consistent spacing
   └─ Result: Footer always at bottom, properly spaced

4️⃣ CSS SELECTOR POWER
   ├─ section + footer (adjacent sibling selector)
   ├─ Selects footer that comes right after section
   ├─ Perfect for styling page footers
   └─ More efficient than class-based selectors
