OPTION 1: FULLY AUTOMATED (Recommended for most users)

On your production server:

    1. SSH to server:
       ssh deploy@your-production-server.com

    2. Navigate to project:
       cd /home/deploy/LMSetjen-DPD-RI

    3. Pull latest code:
       git pull origin main

    4. Run deployment script:
       chmod +x deploy-production.sh
       ./deploy-production.sh

    5. Monitor deployment:
       tail -f backups/deployment_*.log

    Expected time: 15-20 minutes
    ✓ Includes: Backups, migration, health checks


OPTION 2: QUICK MANUAL DEPLOYMENT (For experienced users)
─────────────────────────────────────────────────────────────────────────────

On your production server:

    cd /home/deploy/LMSetjen-DPD-RI

    # Backup current state
    TIMESTAMP=$(date +%Y%m%d_%H%M%S)
    docker compose exec -T postgres pg_dump -U postgres -d lmsetjen_db > \
      backups/db_backup_${TIMESTAMP}.sql

    # Pull latest code
    git pull origin main

    # Rebuild and deploy
    docker compose down
    docker compose build --no-cache backend frontend nginx
    docker compose up -d
    sleep 10

    # Apply migrations
    docker compose exec -T backend python manage.py migrate

    # Verify
    docker compose ps
    curl https://lmsetjendpdri.duckdns.org/login/

    Expected time: 10-15 minutes

✅ POST-DEPLOYMENT VERIFICATION (Must do after deployment)

IMMEDIATELY AFTER DEPLOYMENT (First 5 minutes):

✓ Container Status
  docker compose ps
  # All containers should show "Up"

✓ Check Logs for Errors
  docker compose logs backend | grep -i error
  docker compose logs frontend | grep -i error

✓ Verify Database Connection
  docker compose exec -T backend python manage.py shell << 'EOF'
  from django.contrib.auth.models import User
  print(f"Total users: {User.objects.count()}")
  EOF

WITHIN 15 MINUTES:

✓ Test Web Application
  curl -I https://lmsetjendpdri.duckdns.org/login/
  # Should return: HTTP/1.1 200 OK

✓ Verify SSO Button Visible
  curl -s https://lmsetjendpdri.duckdns.org/login/ | grep -i "nusa\|sso"
  # Should show: Login dengan Nusa DPD

✓ Check Admin Panel
  curl -I https://lmsetjendpdri.duckdns.org/admin/
  # Should return: HTTP/1.1 302 (redirect is OK)

✓ Verify Static Files
  curl -s https://lmsetjendpdri.duckdns.org/static/admin/css/base.css | head -1
  # Should return CSS content, not HTML error

🆘 SOMETHING WENT WRONG? ROLLBACK IN 2 MINUTES
════════════════════════════════════════════════════════════════════════════════

If deployment causes issues:

    cd /home/deploy/LMSetjen-DPD-RI

    # Stop current deployment
    docker compose down

    # Revert to previous commit
    git checkout main~1

    # Rebuild with previous code
    docker compose build --no-cache backend frontend nginx
    docker compose up -d

    # Apply migrations for previous version
    docker compose exec -T backend python manage.py migrate

    # Verify
    docker compose ps

    ✓ System should be back to previous state

If database is corrupted:

    # Restore from backup
    docker compose exec -T postgres psql -U postgres -d lmsetjen_db < \
      backups/db_backup_YYYYMMDD_HHMMSS.sql
