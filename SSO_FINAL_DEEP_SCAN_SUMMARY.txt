🎯 ROUTE FLOW EXPLANATION
════════════════════════════════════════════════════════════════════════════════

HOW REACT ROUTER MATCHES:

User visits: http://localhost:5173/sso/login/abc123

React Router checks each route:
  1. /register/ → No match
  2. /login/ → No match  
  3. /sso/:sso_token/ → No match (more specific route exists below)
  4. /sso/login/:sso_token/ → ✅ MATCH! (renders SSOLogin component)

The "/sso/login/" route must come AFTER "/sso/" in the routes list
because React Router matches routes in order and uses the first match.
