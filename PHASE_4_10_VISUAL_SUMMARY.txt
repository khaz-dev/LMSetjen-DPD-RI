## 🔄 State Flow Diagram

```
USER OPENS EDIT MODAL
         ↓
   openEditModal(user)
         ↓
   API Response:
   {
     full_name: "John",
     email: "john@test.com",
     is_student: true      ←─┐
     is_instructor: true   ←─├─ Extract
     is_admin: false       ←─┘
   }
         ↓
   setFormData({
     ...
     is_student: true      ← Loaded
     is_instructor: true   ← Loaded
     is_admin: false       ← Loaded
   })
         ↓
   FORM DISPLAYS WITH CHECKBOXES
   [✓] Student
   [✓] Instructor
   [ ] Administrator
         ↓
   USER MODIFIES ROLES (e.g., Uncheck Instructor)
         ↓
   setFormData({is_instructor: false})
         ↓
   USER CLICKS SAVE
         ↓
   handleSubmit()
         ↓
   VALIDATION: At least one role? ✅ YES (Student + Admin)
         ↓
   SEND TO BACKEND:
   {
     full_name: "John",
     email: "john@test.com",
     is_student: true,
     is_instructor: false,   ← Changed
     is_admin: false
   }
         ↓
   BACKEND UPDATES USER
         ↓
   SUCCESS: User updated
         ↓
   CLOSE MODAL + REFRESH TABLE
```

## 📊 FormData State Structure

```javascript
INITIAL STATE (Create Mode):
{
  full_name: "",
  email: "",
  role: "student",           ← Legacy field (backward compatibility)
  is_student: false,         ← NEW: Boolean flag
  is_instructor: false,      ← NEW: Boolean flag
  is_admin: false,           ← NEW: Boolean flag
  password: "",
  password2: ""
}

AFTER LOADING USER (Edit Mode):
{
  full_name: "John Doe",
  email: "john@example.com",
  role: "student",           ← Still present
  is_student: true,          ← Loaded from API
  is_instructor: true,       ← Loaded from API
  is_admin: false,           ← Loaded from API
  password: "",
  password2: ""
}

AFTER FORM SUBMISSION:
→ Send to backend:
{
  full_name: "John Doe",
  email: "john@example.com",
  is_student: true,          ← Always included
  is_instructor: true,       ← Always included
  is_admin: false,           ← Always included
  // password only if provided
  // role only in create mode
}
```
