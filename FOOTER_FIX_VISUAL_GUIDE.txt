WHY FRAGMENTS DON'T WORK FOR LAYOUT
═════════════════════════════════════════════════════════════════════════════

                    Fragments vs Divs

┌─────────────────────────────────────────────────────────────┐
│ <> (Fragment)                                               │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ Not rendered as real DOM element!                       │ │
│ │ ❌ Can't apply: display: flex                           │ │
│ │ ❌ Can't apply: flexDirection: column                   │ │
│ │ ❌ Can't apply: minHeight: 100vh                        │ │
│ │ ❌ Children don't get flexbox context                  │ │
│ │ ✅ Good for: conditional rendering, list keys         │ │
│ └─────────────────────────────────────────────────────────┘ │
│ Result: No flex layout, children positioned normally        │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ <div>                                                       │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ Rendered as real DOM container!                         │ │
│ │ ✅ Can apply: display: flex                             │ │
│ │ ✅ Can apply: flexDirection: column                     │ │
│ │ ✅ Can apply: minHeight: 100vh                          │ │
│ │ ✅ Children participate in flexbox layout               │ │
│ │ ✅ Good for: layout containers, grouping content        │ │
│ └─────────────────────────────────────────────────────────┘ │
│ Result: Full flex layout support, footer positioned at bottom
└─────────────────────────────────────────────────────────────┘


FLEXBOX LAYOUT MECHANISM
═════════════════════════════════════════════════════════════════════════════

Step 1: Create Flex Container
  <div style={{ display: 'flex', flexDirection: 'column' }}>
  └─ Makes children stack vertically

Step 2: Set Minimum Height
  minHeight: '100vh'
  └─ Ensures container takes full viewport height

Step 3: Make Content Expandable
  <section style={{ flex: 1 }}>
  └─ Takes all remaining space (after header)
  └─ Prevents footer from being pushed down by content

Step 4: Add Footer Spacing
  CSS rule: margin-top: 80px
  └─ Creates breathing room between content and footer
  └─ Makes visual spacing consistent

Result:
  ┌─────────────────────────────┐
  │ Header (fixed height)       │  ← Fixed position
  ├─────────────────────────────┤
  │ Content                     │
  │ (flex: 1, expands)          │  ← Fills available space
  │                             │
  │                             │
  ├─────────────────────────────┤  ← 80px margin
  │ Footer (with py-3)          │  ← Always at bottom
  └─────────────────────────────┘
