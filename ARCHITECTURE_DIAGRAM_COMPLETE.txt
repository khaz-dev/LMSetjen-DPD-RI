# Multi-Role System - Complete Architecture Diagram

## 1. Authentication Flow

```
┌─────────────────────────────────────────────────────────────┐
│                    USER LOGIN                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  User clicks "Login with Google" or "Nusa DPD"              │
│            ↓                                                  │
│  OAuth provider authenticates user                          │
│            ↓                                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │   Backend /api/v1/auth/google/ or /sso/verify/      │   │
│  ├──────────────────────────────────────────────────────┤   │
│  │ • Verify credentials                                 │   │
│  │ • Generate JWT (includes current_role)              │   │
│  │ • Return user + available_roles                      │   │
│  └──────────────────────────────────────────────────────┘   │
│            ↓                                                  │
│  Frontend receives auth response                            │
│            ↓                                                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │     Check: available_roles.length > 1?              │   │
│  │                                                      │   │
│  │  NO ──→ Direct redirect to dashboard                │   │
│  │  YES → Show RoleSelectionModal                      │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 2. Role Selection Flow

```
┌────────────────────────────────────────────────────┐
│          ROLE SELECTION MODAL                      │
├────────────────────────────────────────────────────┤
│                                                    │
│  👋 Welcome, User Name!                           │
│  user@example.com                                  │
│                                                    │
│  Select your role:                                 │
│  ┌────────────────────────────────────────────┐   │
│  │ 👨‍🎓  Peserta (Student)           [✓]        │   │
│  │      Access as course participant          │   │
│  └────────────────────────────────────────────┘   │
│  ┌────────────────────────────────────────────┐   │
│  │ 👨‍🏫  Instruktur (Teacher)                  │   │
│  │      Access as course instructor           │   │
│  └────────────────────────────────────────────┘   │
│  ┌────────────────────────────────────────────┐   │
│  │ 🛡️   Administrator (Admin)                 │   │
│  │      System administration access          │   │
│  └────────────────────────────────────────────┘   │
│                                                    │
│  ℹ️  You can change roles anytime in settings     │
│                                                    │
└────────────────────────────────────────────────────┘
         ↓
    User selects role
         ↓
    ┌─────────────────────────────────────┐
    │ Call /api/v1/auth/select-role/      │
    │ { role: "teacher" }                 │
    └─────────────────────────────────────┘
         ↓
    Backend updates JWT token
         ↓
    Frontend redirects to dashboard
         ↓
    User sees teacher interface
```

## 3. Permission Checking Flow

```
Request to API endpoint
         ↓
┌────────────────────────────────┐
│   Extract JWT token            │
├────────────────────────────────┤
│ • Access token                 │
│ • current_role = "teacher"     │
│ • user_id = 123                │
└────────────────────────────────┘
         ↓
┌────────────────────────────────────────────────────────┐
│    Permission Check (Phase 2 Classes)                  │
├────────────────────────────────────────────────────────┤
│                                                        │
│  if endpoint requires "teacher" permission:           │
│      ┌─────────────────────────────────────────────┐  │
│      │ Check: current_role == "teacher"?          │  │
│      │                                             │  │
│      │ YES ──→ Allow request ✓                   │  │
│      │ NO  ──→ Deny (403 Forbidden)              │  │
│      └─────────────────────────────────────────────┘  │
│                                                        │
└────────────────────────────────────────────────────────┘
         ↓
   Process or Reject Request
```

## 10. Data Flow Summary

```
User → Login/SSO → Backend Auth → JWT + available_roles
                                          ↓
                                    Frontend Check
                                    ↙            ↘
                            Single Role      Multiple Roles
                                ↓                    ↓
                            Direct Redirect    RoleSelectionModal
                                ↓                    ↓
                            Dashboard          User Selects Role
                                                    ↓
                                            switchRole(role)
                                                    ↓
                                            JWT Token Updated
                                                    ↓
                                            Redirect to Dashboard

Permission Check on Any API Request:
    Extract JWT → Read current_role → Check Permission → Allow/Deny
```
