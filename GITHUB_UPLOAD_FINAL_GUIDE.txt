🆘 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════════

Problem: "Permission denied" when pushing
────────────────────────────────────────────
Solution 1: Use HTTPS with Personal Access Token
  - Generate token at: https://github.com/settings/tokens
  - Use token as password when prompted
  
Solution 2: Use SSH
  - Generate SSH key: ssh-keygen -t ed25519
  - Add to GitHub: https://github.com/settings/keys
  - Use SSH remote: git remote set-url origin git@github.com:YOUR_USERNAME/lmsetjen-dpdri.git

Problem: ".env file was pushed"
────────────────────────────────
Solution: Remove from git history
  git filter-branch --tree-filter 'rm -f .env' -f HEAD
  git push -u origin main --force-with-lease

Problem: Repository is too large
────────────────────────────────
Solution: Check what's tracked
  git ls-tree -r HEAD | sort -k4 -n | tail -20
  
  Large files shouldn't be tracked - check .gitignore

Problem: Can't clone the repository
────────────────────────────────────
Solution: Check repository settings
  - Verify it's public or you have access
  - Verify HTTPS is enabled in Settings > Security
  - Try SSH if HTTPS fails
