TECHNICAL DETAILS
═════════════════════════════════════════════════════════════════════════════

WHY FRAGMENTS DON'T WORK FOR LAYOUTS:

Fragments are special React JSX syntax that don't render in the DOM:

  <> ... </>  ← Renders as: nothing (not even a div!)
  
This means NO container exists to:
  ✗ Apply flexbox properties
  ✗ Set minHeight: 100vh
  ✗ Create flex context for children
  ✗ Position footer correctly

With <div>:
  
  <div style={{display: 'flex', flexDirection: 'column', minHeight: '100vh'}}>
    ↑ Creates actual container
    ├─ Can use flexbox layout
    ├─ minHeight ensures 100vh minimum
    ├─ flex: 1 makes section expand
    └─ Footer always at bottom

WHY 80px MARGIN-TOP MATTERS:

  Admin page structure:
  ┌─────────────────────────────┐
  │ AdminHeader                 │
  │ (fixed position, 60px)      │
  ├─────────────────────────────┤
  │ Section (flex: 1)           │
  │ Content stretches to fill   │
  │ Takes remaining space       │
  ├─────────────────────────────┤  ← margin-top: 80px
  │ Footer (py-3 = 12px padding)│
  │ Consistent 4rem + 80px rule │
  └─────────────────────────────┘

The 80px margin creates breathing room between content and footer,
ensuring visual consistency across all admin pages.

LESSONS LEARNED
═════════════════════════════════════════════════════════════════════════════

1. FRAGMENTS LIMITATION
   Use fragments ONLY for conditional rendering, not layout
   Always use <div> wrapper for page layouts

2. CSS CONSISTENCY
   When fixing one component, check if issue exists elsewhere
   DashboardAdmin became reference template for other admin pages

3. FLEX LAYOUT BEST PRACTICES
   ├─ Root container: display: flex, flexDirection: column, minHeight: 100vh
   ├─ Content: flex: 1 (expands to fill space)
   └─ Footer: margin-top: 80px (consistent spacing)

4. TESTING STRATEGY
   Test all similar pages to catch inconsistencies
   Don't assume other pages are correct without verification
