🚀 HOW IT WORKS

User Login Flow:
────────────────────────────────────────────────────────────────────────────

1. User visits https://lmsetjendpdri.duckdns.org/login/
   └─ Sees new button: "Login dengan Nusa DPD"

2. User clicks SSO button
   └─ Redirected to: https://nusadpd.duckdns.org/

3. User authenticates with Nusa DPD
   └─ Enters NIP and password at SSO provider

4. SSO generates JWT token
   └─ Token contains: nip, name, email, iat, exp

5. Redirects back to LMS with token
   └─ URL: https://lmsetjendpdri.duckdns.org/sso/{token}/

6. Frontend processes token
   └─ Sends: POST /api/v1/sso/verify/ with token

7. Backend verifies token
   └─ Checks JWT signature & expiration
   └─ Looks up or creates user
   └─ Generates LMS JWT tokens

8. Frontend receives tokens
   └─ Stores in secure HttpOnly cookies

9. User redirected to dashboard
   └─ Student → /student/dashboard/
   └─ Teacher → /instructor/dashboard/
   └─ Admin → /admin/dashboard/

10. ✅ Login complete!
