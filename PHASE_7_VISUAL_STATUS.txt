## How It Works

### Role Switching Flow

```
User clicks role badge
       ↓
Dropdown menu opens
       ↓
User selects new role
       ↓
RoleIndicator calls switchRole('new_role')
       ↓
API Request: POST /api/v1/auth/select-role/
       ↓
Backend updates JWT: current_role = 'new_role'
       ↓
Frontend receives success response
       ↓
Toast notification: "Role Switched - You are now in [Role] mode"
       ↓
window.location.reload() [after 500ms delay]
       ↓
User now sees new dashboard with correct role
       ↓
Header displays new role
```

### Component Architecture

```
RoleIndicator (Main)
├─ useRoles() hook (Phase 4)
│  └─ Get: currentRole, availableRoles, rolesLoading
├─ switchRole() util (Phase 4)
│  └─ API call to backend
├─ Toast notifications
│  └─ User feedback
└─ CSS styling
   ├─ Compact badge
   ├─ Expanded card
   ├─ Dropdown menu
   └─ Animations
```
