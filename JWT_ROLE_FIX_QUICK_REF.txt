# QUICK FIX REFERENCE - Role Switching JWT Bug

## The Problem (What You Experienced)
```
Browser Console Error:
- UserData.js:20 role = undefined
- UserData.js:35 role = undefined
- RoleRoute.jsx:84 ❌ Permission denied
```

**Cause:** JWT tokens didn't include role information needed for role-based access control.

---

## The Fix (What Was Changed)

### 1. JWT Token Serializer (serializer.py)
```python
# BEFORE (WRONG)
token['role'] = user.role  # Always 'student' for multi-role users

# AFTER (FIXED)
token['role'] = user.current_role if hasattr(user, 'current_role') and user.current_role else user.role
token['current_role'] = user.current_role if hasattr(user, 'current_role') else user.role
```

### 2. Token Generation Helper (views.py)
```python
def generate_tokens_with_role(user):
    """Generate tokens using serializer to include role info"""
    refresh = RefreshToken.for_user(user)
    MyTokenObtainPairSerializer._add_user_fields(refresh, user)
    MyTokenObtainPairSerializer._add_user_fields(refresh.access_token, user)
    return {
        'access_token': str(refresh.access_token),
        'refresh_token': str(refresh)
    }
```

### 3. Role Switch Endpoint (views.py - SelectRoleAPIView)
```python
# BEFORE (WRONG)
refresh = RefreshToken.for_user(user)  # Missing role info

# AFTER (FIXED)
tokens = generate_tokens_with_role(user)  # Includes role info
```
