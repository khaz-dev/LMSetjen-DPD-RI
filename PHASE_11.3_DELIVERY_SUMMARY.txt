🎨 USER EXPERIENCE FLOW
────────────────────────────────────────────────────────────────────────────────

    1️⃣  User Fills Form
        [Title] [Description] [Area] [URL]
                    ↓
    2️⃣  Clicks "✓ Kirim Masukan"
                    ↓
    3️⃣  Frontend Validates
        ✓ Title: 3+ chars
        ✓ Description: 10+ chars
        ✓ URL format valid
                    ↓
    4️⃣  VERIFICATION MODAL APPEARS
        ┌─────────────────────────────────┐
        │ 🔐 Verifikasi Masukan Anda    ✕ │
        │                                 │
        │ Ringkasan Masukan Anda:         │
        │ [Feedback Summary Card]         │
        │                                 │
        │ Harap klik poin-poin secara     │
        │ berurutan:                      │
        │                                 │
        │   ⭘    ⭘    ⭘    ⭘           │
        │   1    2    3    4             │
        │                                 │
        │ Klik dimulai dari poin 1       │
        │                                 │
        │ [Batal] [✓ Konfirmasi] ❌      │
        └─────────────────────────────────┘
                    ↓
    5️⃣  User Clicks Dots in Sequence
        • Click Dot 1: ⭘→✓, Dot 2 pulses "🎯 Bagus! Lanjutkan ke 2"
        • Click Dot 2: ⭘→✓, Dot 3 pulses "⚡ Sempurna! Klik 3"
        • Click Dot 3: ⭘→✓, Dot 4 pulses "🔥 Hampir selesai! Klik 4"
        • Click Dot 4: ⭘→✓, Success! "🎉 Verifikasi lengkap!"
                    ↓
    6️⃣  Submit Button Enabled
        "✓ Konfirmasi & Kirim Masukan" button becomes bright/enabled
                    ↓
    7️⃣  User Clicks "Konfirmasi & Kirim"
        Button shows: "⟳ Mengirim..."
                    ↓
    8️⃣  Backend Processes
        POST /feedback/create/
        Captures: user_role_at_submission (immutable)
                    ↓
    9️⃣  Success! ✅
        Toast: "Terima Kasih! Masukan Anda telah berhasil dikirim"
        Modal closes
        Form resets


🔐 ANTI-SPAM FEATURES
────────────────────────────────────────────────────────────────────────────────
✅ Sequence Verification     → Dots must be clicked in exact order
✅ Manual Engagement         → User must actively interact (not automated)
✅ Form Validation           → Multiple validation layers
✅ Backend Validation        → Double-check on server side
✅ Role Immutability         → Role captured at submission time
✅ Rate Limiting Ready       → Backend can track attempts (future)
