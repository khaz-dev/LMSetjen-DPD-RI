ISSUE: Restrictive Nginx Regex Pattern
─────────────────────────────────────────

Original Nginx Configuration:

  location ~ ^/admin/(dashboard|users|courses|analytics|reports|system|profile)/ {
      try_files $uri $uri/ /index.html;
  }
  
  location /admin {
      proxy_pass http://lms_backend:8000;
  }

Problem:
  • The regex pattern ONLY matched specific routes
  • Routes included: dashboard, users, courses, analytics, reports, system, profile
  • Routes MISSING: kelola-materi ❌
  
  When accessing: /admin/kelola-materi/
  
  1. Nginx checks if matches regex pattern
  2. Pattern has hardcoded route list: (dashboard|users|courses|...)
  3. "kelola-materi" NOT in list
  4. Falls through to /admin proxy
  5. Proxies to Django admin backend
  6. Django requires authentication
  7. Returns 302 redirect to login page

Affected Routes:
  ✅ /admin/dashboard/     (in whitelist)
  ✅ /admin/users/         (in whitelist)
  ❌ /admin/kelola-materi/ (NOT in whitelist)
  ✅ /admin/documentation/ (might be in whitelist)

Why Localhost Worked:
  • Vite dev server (http://localhost:5173) doesn't have nginx
  • Vite serves React SPA directly
  • React Router handles all routes client-side
  • No nginx restrictions

FIXED NGINX CONFIGURATION
──────────────────────────

Changed from restrictive regex to simple prefix match:

  # React Admin Routes - Now catches /admin/kelola-materi/ and all others
  location /admin/ {
      try_files $uri $uri/ /index.html;
      add_header Cache-Control "no-cache, no-store, must-revalidate" always;
      add_header Pragma "no-cache" always;
      add_header Expires "0" always;
  }

Benefits:
  ✅ Matches ALL routes: /admin/*
  ✅ No hardcoded route list
  ✅ React SPA fallback works for all routes
  ✅ Future admin routes automatically supported
  ✅ No nginx changes needed when adding new pages

Route Handling Flow:
  
  /admin/kelola-materi/
         ↓
  nginx matches: location /admin/
         ↓
  try_files $uri $uri/ /index.html
         ↓
  File not found → serves index.html (React SPA)
         ↓
  React Router receives URL
         ↓
  React loads KelolaMaterialAdmin component
         ↓
  ✅ Page displays correctly!

NGINX LOCATION BLOCK PRECEDENCE:
────────────────────────────────

  1. Exact match (=)
  2. Longest prefix match
  3. Case-insensitive regex (~*)
  4. Case-sensitive regex (~)
  5. Prefix match ← We use this

Our /admin/ block:
  • Uses prefix match (lowest precedence)
  • Matches: /admin/kelola-materi/, /admin/dashboard/, /admin/users/, etc.
  • Executes: try_files (React SPA fallback)
  • Result: Serves index.html for all unmatched paths
