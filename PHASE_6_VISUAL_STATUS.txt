## How It Works

### Route Access Flow

```
User tries to access /instructor/dashboard/
         ↓
<Route path="/instructor/dashboard/">
  <PrivateRoute>              ← Step 1: Check authentication
    <RoleRoute allowedRoles={["teacher", "instructor"]}>
                              ← Step 2: Check role permission
      <Dashboard />           ← Step 3: Render component
    </RoleRoute>
  </PrivateRoute>
</Route>

Step 1: PrivateRoute checks isLoggedIn()?
├─ NO:  Redirect to /login/
└─ YES: Continue to RoleRoute

Step 2: RoleRoute checks currentRole in allowedRoles?
├─ Loading: Show "Verifying access..."
├─ YES:    Render Dashboard ✓
└─ NO:     Show error toast + Redirect to / ✗

Step 3: Dashboard renders if authorized
        Show instructor interface
```

### Multi-Role User Switching Roles

```
User at /instructor/dashboard/ (currentRole = "teacher")
         ↓
Click header role switcher (Phase 7 feature)
         ↓
Select "Student" role
         ↓
switchRole("student") API call
         ↓
JWT token updated: current_role = "student"
         ↓
RolesContext updated
         ↓
RoleRoute re-evaluates: "student" in ["teacher", "instructor"]? NO
         ↓
Show error: "You don't have permission..."
         ↓
Redirect to /
         ↓
User can now access /student/dashboard/ ✓
```
